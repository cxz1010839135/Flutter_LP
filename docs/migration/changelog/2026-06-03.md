## Windows MSI 打包链路

| 环节 | 文件/目录 | 说明 |
|------|-----------|------|
| 双击入口 | `打包Windows安装包.bat` | 检查 `pubspec.yaml`、`flutter`、`dotnet`；路径用 `%~dp0` |
| 主脚本 | `scripts/package/windows.ps1` | `$PSScriptRoot` 定位工程根，与 cwd 无关 |
| 执行策略 | `scripts/package/windows.cmd`、`scripts/build_msi.cmd` | `-ExecutionPolicy Bypass` 调用 ps1 |
| WiX 工程 | `installer/LPRobot.Installer.wixproj` | WiX SDK 6.x + Heat；`EmbedCab` 单文件 MSI |
| 安装定义 | `installer/wix/Package.wxs` | 产品名「领鹏智能」、`WixUI_InstallDir`、快捷方式 |
| 产物 | `dist/LPRobot-<版本>-x64.msi` | 版本来自 `pubspec.yaml` |

### 打包脚本自动步骤

```text
flutter build windows --release  →  领鹏智能.exe
同步 config/  →  Release/config/
同步 dll/visualprogram/  →  Release/dll/visualprogram/  （仅 Blockly 运行资源）
删除 installer/obj、build/installer  （避免 Heat 引用已删文件）
dotnet build installer/LPRobot.Installer.wixproj
复制 MSI  →  dist/
```

日志成功标志：`staged xxxx files under dll/visualprogram/`（xxxx > 0）。

## 发布 exe 与路径

- `windows/CMakeLists.txt`：`BINARY_OUTPUT_NAME = 领鹏智能`；目标名保持 `flutter_application_1`（Flutter 工具链要求）
- `windows/runner/CMakeLists.txt`：`OUTPUT_NAME` 映射到 `BINARY_OUTPUT_NAME`
- `lib/core/robot_path_layout.dart`：`windowsReleaseExeName`、`windowsExeNames`（兼容 Debug 旧名）
- `lib/platform/robot_paths_windows.dart`：探测安装根；不可写时回退 `%LOCALAPPDATA%\Lingpeng\LPRobot\` 并 seed 默认配置

---

## dll 目录约定（重要）

- 运行时只读 **`dll/visualprogram/`**（见 `lib/blockly/lp_blockly_config.dart`）
- **不要**在 `dll/` 根目录放 `.zip`、`.deb`、`.lnk`、其它版本备份文件夹
- 修改 Blockly 资源后须 **完整双击 bat**，勿仅用 `-SkipFlutterBuild`（除非刚完整 build 且 Release 已同步）

### Blockly 对齐范围（已验证）

与 Android `ProgramActivity` / `BoundObject` 一致的操作包括：

- 本地 HTTP 加载 `dll/visualprogram`（shelf + WebView）
- 读写 `config/server`（main.xml / main.rp4）、`files/xml`、`files/projects`、`files/funlib`
- 工具栏保存、编译结果、选 XML 加载
- 退出流程：离线写 server；在线上传 + 失败重传对话框
- 加载/保存进度遮罩（Windows WebView 隐藏原生层）

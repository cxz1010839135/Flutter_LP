# 2026-06-16 变更详情（1.7.1）

## 背景

对齐驱控 **D9000 / initstatus** 状态含义；明确维护区在**程序运行中**与**已停止**下的可操作范围，避免运行中误改驱控文件或误入调试模式。

---

## 一、启动状态（D9000 对照表）

HTTP 轮询字段 `initstatus` 用于底栏/连接面板「启动状态」（非监控页手动读寄存器 D9000，但语义与 D9000 插补状态表一致）。

| 值 | 含义 | 展示色 |
|----|------|--------|
| `0` | 空闲中 | 正常 |
| `1` | 队列等待执行 | 正常 |
| `2` | 执行中（等料） | 正常 |
| `3` | 运动执行中 | 正常 |
| `< 0` | 按表报错（如 `-5` 不支持指令） | 报警 |

### 涉及文件

| 文件 | 说明 |
|------|------|
| `lib/core/robot_d9000_status.dart` | D9000 对照表、`formatInline` / `formatStatusLine` |
| `lib/core/robot_alarm_info.dart` | `formatInitStatus`、`initStatusOk`（≥0 为正常） |
| `lib/app/widgets/lp_robot_foot_bar.dart` | 底栏启动状态展示 |
| `lib/app/widgets/lp_robot_alarm_status_bar.dart` | 报警状态条 |
| `lib/features/monitor/monitor_d9000_status.dart` | 监控页别名，复用核心表 |

**说明**：电机报警仍走驱控 `alarm` / `alarmCode` 位掩码表，与启动状态分开解析。

---

## 二、维护区运行门控

原则：**可随时进入维护查看**；**仅在程序已停止时**允许修改驱控文件或进入调试相关功能。

判定：`MaintenanceEditGate.canEdit()` → `!RobotTelemetry.isRobotMoving`（与主页右侧「启动 / 停止」一致）。

### 程序运行中（`isRobotMoving = true`）

| 位置 | 行为 |
|------|------|
| 文件配置 | 隐藏：保存文件、创建该文件、配置扩展、添加/删除条目；点击表格行 → 只读查看弹窗 |
| 文件配置底栏 | 「调试模式」灰色不可点 |
| 文件管理 | 隐藏：上传到驱控、一键备份、一键恢复；可浏览目录、点文件预览、下载 |
| 版本页（ToolPage） | 禁用：打开/关闭调试模式、驱动器参数、机代码自动运行相关按钮 |

### 程序已停止（`isRobotMoving = false`）

上述修改类按键与「调试模式」入口全部恢复可用。

### 涉及文件

| 文件 | 说明 |
|------|------|
| `lib/core/maintenance_edit_gate.dart` | 统一门控 |
| `lib/features/config_file/config_file_page.dart` | 配置向导按键显隐、只读查看 |
| `lib/features/files/files_page.dart` | 上传/备份/恢复显隐、驱控文件预览 |
| `lib/features/tool/tool_page.dart` | 版本页调试相关按键 |
| `lib/features/home/main_home_page.dart` | 维护导航：已连接即可进入 |
